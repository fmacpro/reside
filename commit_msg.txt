Fix parser JSON repair for template literals with ${} interpolation

The repairJson() function had a bug where it tried to escape ${} markers
inside backtick template literals. The replacement string was being
interpreted by JavaScript's regex replacement engine — $ has special
meaning in replacement strings ($&, $`, $'), so it produced ${ instead
of \${, causing JSON.parse() to fail with 'Bad escaped character'.

The fix: ${} is valid inside JSON string values and doesn't need escaping.
Only the backtick delimiter needed conversion to double quotes.
